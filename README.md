# Учёт заказов — Telegram-бот на Vercel

Считает продажи по заказам из Telegram. Каждый заказ = список товаров с ценами +
доставка (1500 тг по городу, 2000 тг за городом — сумма не зависит от количества
товаров в заказе). Итоги считаются по календарным дням (сброс в полночь по
Asia/Almaty), историю можно посмотреть за любую прошедшую дату.

## Как это работает

- **Бот** — основной способ ввода. Пишете сообщение с товарами и городом, бот
  считает и сохраняет заказ.
- **Веб-страница `/api/report`** — показать итоги в браузере, без Telegram.
- **База данных** — Postgres на [neon.tech](https://neon.tech) (таблицы создаются
  автоматически при первом обращении, вручную ничего создавать не нужно).

## Формат сообщения боту

```
Кроссовки 15000
Футболка 5000
город
```

Каждая строка — товар и цена. Последняя (или любая) строка — `город` или
`не город` (можно также: `г`, `нг`, `загород`, `вне города`). Доставка
добавится автоматически.

## Прайс-лист (необязательно)

Чтобы не вписывать цену каждый раз для одних и тех же товаров, один раз
сохраните прайс командой `/price` (каждая строка — «Название цена»):

```
/price
Waka 10 13000
Rif 15000
Waka 8 12500
Elf 25 13500
Elf 30 14500
Drag 12500
Plus 11000
```

После этого в заказе достаточно написать только название товара — цена
подставится сама:

```
Waka 10
Elf 25
город
```

Если для конкретного заказа нужна другая цена — просто укажите её вручную
(`Waka 10 12000`), она перекроет цену из прайса. `/prices` показывает
сохранённый прайс, повторный `/price` с тем же названием обновляет цену.

## Команды бота

- `/help` — справка
- `/today` — итоги за сегодня
- `/date 21.07.2026` — итоги за конкретную дату
- `/undo` — удалить последний добавленный заказ (если ошиблись)
- `/price` — сохранить/обновить прайс-лист
- `/prices` — показать сохранённый прайс-лист

## Деплой на Vercel

1. Залейте этот репозиторий на GitHub.
2. На [vercel.com](https://vercel.com) → **Add New → Project** → выберите репозиторий.
   Framework Preset можно оставить "Other" — Vercel сам подхватит папку `api/`.
3. **Создайте базу данных на [neon.tech](https://neon.tech)** (напрямую, в обход
   Vercel Storage/Marketplace — так надёжнее): Sign in → Continue with GitHub →
   Create a project. На странице проекта скопируйте **Connection string**
   (Connection Details). В Vercel → **Settings → Environment Variables**
   добавьте `DATABASE_URL` = эта строка целиком.
4. Создайте бота через [@BotFather](https://t.me/BotFather) (`/newbot`), получите токен.
5. В настройках проекта Vercel → **Settings → Environment Variables** добавьте:
   - `TELEGRAM_BOT_TOKEN` — токен от BotFather
   - `TELEGRAM_WEBHOOK_SECRET` — любая случайная строка (придумайте сами)
6. Задеплойте (Deploy). После деплоя откройте один раз в браузере:
   ```
   https://<ваш-проект>.vercel.app/api/setup?secret=<TELEGRAM_WEBHOOK_SECRET>
   ```
   Это зарегистрирует вебхук бота. В ответ должно прийти `"ok": true` от Telegram.
7. Готово — пишите боту в Telegram.

## Локальная разработка (опционально)

```
npm install
npm i -g vercel
vercel dev
```

Понадобится `vercel env pull`, чтобы получить переменные окружения локально.

## Настройки тарифов

Необязательные переменные окружения (по умолчанию уже стоят нужные значения):

- `CITY_DELIVERY_FEE` — доставка по городу (по умолчанию `1500`)
- `OUTSIDE_CITY_EXTRA_FEE` — доплата за город (по умолчанию `500`, итого `2000`)
- `TIMEZONE` — часовой пояс для суточного среза (по умолчанию `Asia/Almaty`)
